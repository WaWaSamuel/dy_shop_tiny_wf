当前前端页面情况

DYShop 前端页面结构图

这是根据当前项目中的真实路由、布局和页面组件整理出来的页面全景图。 结论上，这个项目目前是一个“个人结果展示台”，核心由主页、资讯结果、执行记录中心，以及一个包含多个子页的电商工作台组成。

框架：Vite + React + TypeScript + Ant Design 路由：React Router v6 主视觉：贴纸风 / 卡片化 / 看板式信息组织 真实可访问主页面：7 个

个人结果展示台

`/`

首页总入口。以项目卡片方式进入不同结果模块，并展示宿主登录态、主入口状态和快捷概览。

项目卡片导航 登录态监控 首页总览

资讯聚合引擎

`/project/news`

资讯日报结果页。展示信息源状态、热点提炼、文章流、推送记录，并支持推送到飞书。

资讯结果展示 热点聚合 文章详情抽屉

执行记录中心

`/project/runtime`

查看 agent、workflow、skill 的目录和执行记录，包含筛选、统计卡片、能力表格和最近记录列表。

日志中心 能力目录 记录筛选
↓ 电商工作台分支
ProjectLayout
`/project/ecommerce/*`
固定左侧栏 + 工作台内容区

这是项目最完整的一条业务线，左侧栏固定挂载 5 个子入口：总览、选品、货盘、素材工坊、流程看板。

电商结果总览

`/project/ecommerce/overview`

电商模块的主入口页，强调“先看结果，再决定下一步”。展示结果快照、推荐关注项、工作对象表格和阶段分布。

结果摘要 推荐关注 跳转货盘/轨迹

抖掌柜货盘

`/project/ecommerce/products`

货盘读取页。支持导入 Excel、搜索筛选、守门状态查看、结果快照查看，并从每个商品跳到流程轨迹。

Excel 导入 表格结果页 守门状态

候选结果快照

`/project/ecommerce/sourcing`

选品对比页。以候选商品卡片和推荐分方式展示“谁更值得推进”，偏结果判断而非实际执行。

来源 Tabs 商品卡片流 推荐快照

素材工坊

`/project/ecommerce/creative-studio`

素材结果回流页。左侧是筛选条件与同步入口，右侧是素材版本瀑布和底部时间线。

版本卡片 详情描述 结果时间线

流程看板

`/project/ecommerce/flow/:id`

单商品维度的流程轨迹页。用 React Flow 把各阶段节点画成流程图，点击节点会打开右侧详情抽屉。

流程图画布 节点抽屉 进度概览

页面层次总结

现在的页面组织逻辑
  • 第一层是“首页总入口”，用项目卡片来分发不同工作台。
  • 第二层是“独立结果页”，包括资讯聚合与执行记录中心。
  • 第三层是“电商工作台子系统”，内部再细分成总览、货盘、选品、素材、流程轨迹。
  • 整体不是传统后台的 CRUD 结构，而是“结果展示台 + 回看分析台”。
最核心的用户流
  • 从首页进入电商工作台。
  • 先在总览页看“当前最值得复看什么”。
  • 进入货盘页查看具体货品和结果阶段。
  • 从货品进入流程看板，追踪单个商品的阶段详情。
  • 如果要看候选结果或素材回流，再分别进入选品页和素材工坊。
